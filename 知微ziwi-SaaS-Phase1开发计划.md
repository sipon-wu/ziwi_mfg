## 二、架构技术栈

| 层 | 技术选型 | 说明 |
|----|---------|------|
| 前端 | Vue 3 + Vant（移动端）+ DataV/ECharts（大屏） | 与能碳系统一致 |
| 后端 | FastAPI（Python 3.13） | 与能碳系统一致 |
| 数据库 | PostgreSQL + TimescaleDB | 多租户 tenant_id 隔离 |
| 连接池 | PgBouncer（事务池模式） | 150 连接支撑 1,000 租户 |
| 缓存 | Redis | 会话/工价/权限缓存 |
| 消息 | 企业微信 + 钉钉 消息推送 | Phase 1 仅推送，不做审批双向 |
| 部署 | Docker Compose + Nginx | SaaS + 本地部署统一编排 |
| IoT | 边缘网关（72h 缓存 + 自动重传） | M12 数据采集 |

## 三、开发任务分解

### 3.1 M0 — 基础平台（W1~W3，共 9 个任务）

#### [ ] 任务 1：项目脚手架搭建
**描述**：初始化 Vue 3 + FastAPI 项目结构，Docker Compose 编排
**验收条件**：
- 前端项目可 `npm run dev`，后端可 `uvicorn` 启动
- Docker Compose 一键启动 PostgreSQL + Redis + PgBouncer
- 前端/后端项目结构对齐能碳系统既有规范
- 健康检查端点 `/health` 返回 200
- 首屏 1.5 秒内加载
**ADR 参考**：ADR-001（DAL 抽象层）

#### [ ] 任务 2：多租户数据访问层（DAL）
**描述**：实现 Repository 模式 DAL，自动注入 tenant_id
**验收条件**：
- `MultiTenantRepository` 基类自动附加 `WHERE tenant_id = :tenant_id`
- SaaS 模式通过 JWT 解析 tenant_id，本地模式通过配置注入
- 支持 PostgreSQL 行级安全策略（RLS）作为第二道防线
- 代码覆盖测试 > 80%
**ADR 参考**：ADR-001、ADR-002（PgBouncer 事务池）

#### [ ] 任务 3：用户与权限系统
**描述**：租户管理、用户体系、RBAC 权限、密钥管理
**验收条件**：
- 租户注册（SaaS 自注册）/ 创建（本地部署管理员创建）
- 用户登录：密码 + 验证码
- RBAC 三级：角色 → 模块权限 → 功能权限
- 平台功能（用户管理/许可证查看）永不限制，预埋离线降级逻辑
- 密钥管理预埋 RSA-2048 密钥对生成逻辑

#### [ ] 任务 4：消息中心
**描述**：站内信 + 企业微信/钉钉消息推送集成
**验收条件**：
- 站内信：发送/接收/已读/未读计数
- 企业微信消息推送：文本消息 + 卡片消息
- 钉钉消息推送：工作通知消息
- 消息模板管理
**ADR 参考**：Phase 1 仅推送，审批双向为 Phase 2

#### [ ] 任务 4b：审批引擎（内置审批流）
**描述**：实现知微内置审批流引擎，支持审批模板配置、审批流程流转、审批意见记录。Phase 1 使用内置审批流，不依赖 OA 系统。
**验收条件**：
- 审批模板管理：CRUD 审批模板，支持多步骤审批节点配置（会签/或签）
- 审批人配置：支持按角色指定（部门主管/指定角色/指定用户）
- 审批发起：业务系统通过 API 调用发起审批，返回审批单 ID
- 审批处理：审批人可审批通过/驳回/转审，支持填写审批意见
- 审批通知：审批待办通过消息中心（站内信 + 企微/钉钉）推送给审批人
- 审批状态查询：查询审批单当前状态、审批历史流转记录
- 审批超时提醒：审批节点超时（可配置，默认 48h）自动触发消息催办
- 数据隔离：审批模板和审批单均按 tenant_id 隔离
**依赖关系**：依赖 任务 3（用户与权限系统）+ 任务 4（消息中心）
**ADR 参考**：无独立 ADR，遵循产品方案 §9.1 / §10.2 内置审批流策略

#### [ ] 任务 5：数据字典与基础配置
**描述**：全局字典、车间/线别/工站/工序配置、班组管理、员工档案
**验收条件**：
- 字典管理（枚举值维护，如不良原因、异常类型）
- 车间 - 线别 - 工站 - 工序 四级配置
- 班组管理（CRUD + 成员关联）
- 员工档案（技能等级、所属班组）
- 工厂类型配置（离散型/流程型/混合型）——预埋，Phase 1 仅离散型

#### [ ] 任务 6：许可证管理（本地部署）
**描述**：RSA 签名激活码 + 硬件指纹 + 混合验证状态机
**验收条件**：
- 在线验证：心跳请求 `POST /v1/verify`
- 离线验证：本地签名验证 + 硬件指纹比对
- 功能限制：仅 2 状态（正常/限制），单阈值（30 天）
- 离线计时：`time.monotonic()` + 云端比对，不加密不 checksum
- 预警：
  - 第 1 天触发：邮件 + 站内信（无短信）
  - 第 7 天触发：邮件 + 短信 + 站内信
  - 第 21 天触发：邮件 + 短信 + 站内信
  - 第 30 天触发：邮件 + 短信 + 客服外呼
**ADR 参考**：ADR-006（离线激活）、ADR-020（离线计时）

#### [ ] 任务 7：PgBouncer 连接池配置
**描述**：部署配置 PgBouncer 事务池模式
**验收条件**：
- 事务池模式（`pool_mode = transaction`）
- 支持 150 个客户端连接 → 150 个池连接 → 150 DB 连接
- 空闲超时和查询超时配置
- 多租户下的连接复用验证
**ADR 参考**：ADR-002

#### [ ] 任务 7B：API 网关熔断器（TenantCircuitBreaker）
**描述**：实现多租户 API 网关熔断中间件，基于滑动窗口 P95 延迟 + 错误率触发断路器状态机，配合紧急白名单机制保障核心链路。
**验收条件**：
- 滑动窗口指标收集：每租户独立维护 60 秒滑动窗口，统计 P95 延迟（ms）和错误率（%），窗口粒度 1 秒桶
- 断路器状态机：CLOSED → OPEN → HALF_OPEN → CLOSED 三态转换
  - CLOSED→OPEN：P95 > 2000ms 或 错误率 > 10% 持续 10 秒
  - OPEN→HALF_OPEN：OPEN 持续 30 秒后自动半开（允许 1 个探测请求）
  - HALF_OPEN→CLOSED：探测请求成功；→OPEN：探测请求失败
- 紧急白名单：可配置白名单 URL 前缀（如 `/health`, `/v1/verify`, `/v1/activate`），跳过熔断检测
- FastAPI 中间件集成：以 `@app.middleware("http")` 或 ASGI 中间件形式注入，自动按 `X-Tenant-ID` 请求头提取租户上下文
- 熔断事件上报：熔断状态变更时，通过消息中心发送告警，含租户 ID、当前状态、触发指标
- 熔断指标 API：`GET /admin/circuit-breaker/status` 返回概览；`GET /admin/circuit-breaker/{tenant_id}` 返回单租户指标
- 配置热加载：熔断阈值支持从配置中心动态更新，无需重启服务
- 熔断 OPEN 时返回 503 + `{"code":"CIRCUIT_OPEN","message":"service temporarily unavailable"}`
**依赖关系**：依赖 任务 1（脚手架基础框架）；不阻塞其他任务，作为网关中间件自动生效
**ADR 参考**：ADR-003（多租户负载熔断）

### 3.2 M1 — 数据采集模块（W3~W5，共 4 个任务）

#### [ ] 任务 8：IoT 数据采集网关
**描述**：边缘网关架构，支持 MQTT/Modbus/OPC UA
**验收条件**：
- MQTT Broker 接入（支持 EMQX/VerneMQ）
- Modbus TCP/RTU 接入（支持主流 PLC）
- OPC UA 客户端接入
- 数据校验：格式校验 + 范围校验 + 时间戳校验
- 网关配置（采集频率、点位映射、上下限）
- 72 小时本地边缘缓存 + 自动重传 + 去重
**ADR 参考**：ADR-019（72h 缓存 + 自动重传 + 去重）

#### [ ] 任务 9：Excel 导入引擎
**描述**：批量导入模板引擎，验证规则
**验收条件**：
- 8 个行业导入模板（生产/TPM/品质/安灯/能碳等）
- 字段校验（必填/格式/范围/唯一性）
- 批量导入性能：10 万行 < 30 秒
- 导入结果反馈（成功/失败/警告行级明细）
- 模板下载功能

#### [ ] 任务 10：设备数据管理
**描述**：设备注册、测点管理、采集数据存储
**验收条件**：
- 设备注册（设备 ID/名称/类型/协议/采集配置）
- 测点管理（测点名称/单位/精度/上下限）
- TimescaleDB 超表存储时序数据
- 数据查询 API（按设备/测点/时间范围）

#### [ ] 任务 11：数据链路监控
**描述**：采集通道健康监控，断线告警
**验收条件**：
- 设备在线/离线状态检测
- 数据采集通道健康仪表盘
- 断线告警（企业微信/钉钉推送）
- 数据延迟告警（超过 N 分钟未收到数据）
