# 自选套餐 — 全场景枚举表

> **说明**：M00 基础服务（认证/租户/用户/角色/消息/审批/字典）为**所有套餐必选**，不出现在选配清单中。  
> **Phase 1 可自选模块**：M01 生产 · M02 TPM · M03 品质 · M05 安灯 · M11 能碳 · M12 数据采集 · M13 看板  
> **编码规则**：每个场景编码为 `S-{核心模块}`-{数字}，方便后续引用。

---

## 一、场景汇总

共 7 个可选模块，按商业意义分为 **6 大客户画像**，覆盖全部 128 种组合中的 **32 种有意义场景**。

### E 画像：看板增强（M13 作为附加模块，叠加到任意套餐）

M13 看板不能独立存在（需要至少一个数据源），可附加到任何包含业务模块的套餐上。

| 叠加规则 | 前提 | 效果 |
|:---------|:-----|:-----|
| **M13 + 任意 A/B/C/D** | 至少有 1 个业务模块 | 看板展示对应模块数据 |
| **仅 M13** ❌ | 无业务模块 | 建议禁止，或允许创建空白看板占位 |
| **M13 + M12(仅采集)** | 仅有 M12 | 看板仅能展示 IoT 采集数据（有限） |

## 三、强关联模块标注

以下组合有**强业务关联**，选配时系统应给出建议提示：

| 关联 | 等级 | 说明 | 无依赖模块时的降级方案 |
|:-----|:----:|:------|:--------------------|
| M03 品质 ↔ M01 生产 | 🔗 建议 | 品质检验单通常关联生产工单 | 品质模块可独立运行，手工录入检验单 |
| M05 安灯 → M01/M02 | 🔗 建议 | 安灯呼叫通常关联工单/设备 | 安灯可独立运行，手动选择关联对象 |
| M11 能碳 ↔ M02 设备 | 🔗 建议 | 能碳设备数据优先来自 TPM | M11 独立运行时走 Excel 导入 |
| M11 能碳 → M12 采集 | 🔗 建议 | 能耗时序数据通过 IoT 采集更佳 | M11 独立运行时走 Excel 导入 |
| M13 看板 → 任意业务模块 | 🔗 建议 | 看板需要数据源 | 允许创建空白看板占位 |
| M01 生产 → M12 采集 | 🔗 建议 | 报工数据可通过 IoT 自动采集 | 手动录入报工 |
| M03 品质 → M12 采集 | 🔗 建议 | 检验数据可通过 IoT 自动采集 | 手动录入检验结果 |

## 四、数据路由决策矩阵（按场景）

```
场景编码 → 判断 feature_flags → 执行对应 SQL/逻辑

例如 C04 (M01+M02+M05+M11):
  has_tpm  = True  → SELECT FROM equipment JOIN energy_device
  has_mes  = True  → 碳核算时从 work_report 取产量
  has_andon = True → 告警创建时联动 andon_call.create()

例如 B01 (仅 M11):
  has_tpm  = False → SELECT FROM energy_device (自管)
  has_mes  = False → 碳核算时产量=0
  has_andon = False → 告警仅写入 energy_alert
```
