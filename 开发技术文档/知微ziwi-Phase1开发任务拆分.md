# 知微 SaaS Phase 1 — 开发任务拆分

> **依据**：Phase 1 开发计划（已修订版，34 个任务）  
> **日期**：2026-06-12  
> **优先顺序**：P0 → P1，P0 为编码启动必须完成的任务

---

## P0 — 编码启动必备（共 13 人日）

### Story 1：用户认证 + 租户管理 + 权限基础（3人日 · 后端）

**对应开发计划任务**：任务 1（脚手架）+ 任务 2（DAL）+ 任务 3（用户权限）

| 子任务 | 估时 | 产出物 |
|--------|:----:|--------|
| 项目脚手架搭建（Vue 3 + FastAPI + Docker Compose） | 0.5人日 | 项目骨架、docker-compose.yml |
| 多租户 DAL 层实现（Repository 模式 + tenant_id 自动注入） | 1人日 | DAL 基类、SaaS/Onprem 实现 |
| 用户注册/登录/Token 管理 | 0.5人日 | JWT 认证中间件、登录/刷新 API |
| 租户 CRUD + 配置管理 | 0.5人日 | 租户管理 API |
| RBAC 权限（角色→模块→功能三级） | 0.5人日 | 角色/权限 CRUD API |

**验收标准**：
- `POST /api/v1/auth/login` 返回有效 JWT Token
- `GET /api/v1/tenants` 返回租户列表（分页）
- `GET /api/v1/users/me` 返回当前用户信息+角色
- DAL 层自动注入 `tenant_id`，跨租户隔离验证通过
- 健康检查 `/health` 返回 200

### Story 2：M12 数据采集（Excel 导入）（3人日 · 后端）

**对应开发计划任务**：任务 9（Excel 导入引擎）

| 子任务 | 估时 | 产出物 |
|--------|:----:|--------|
| 导入模板设计与下载接口 | 0.5人日 | 8 个行业模板文件 |
| Excel 解析引擎（openpyxl + 校验规则） | 1人日 | 字段校验（必填/格式/范围/唯一性） |
| 异步导入 + 批量写入 DB | 1人日 | 10 万行 < 30 秒 |
| 导入结果反馈（成功/失败/警告行级明细） | 0.5人日 | 导入结果 API + 错误行下载 |

**验收标准**：
- 模板下载：`GET /api/v1/excel-import-tasks/templates/{type}`
- 导入提交：`POST /api/v1/excel-import-tasks`
- 10 万行数据导入 < 30 秒
- 校验失败行有明确的错误提示（字段名+原因）

### Story 3：M01 生产管理 — 报工基础版（5人日 · 后端+前端）

**对应开发计划任务**：任务 12（任务单）+ 任务 14（个人报工）+ 任务 15（报工报表）

| 子任务 | 估时 | 产出物 | 角色 |
|--------|:----:|--------|:----:|
| 工单 CRUD + 状态流转 | 1人日 | 工单管理 API | 后端 |
| 个人报工提交 + 任务单回写 | 1人日 | 报工 API | 后端 |
| 报工报表（日报/月报） | 1人日 | 报表查询 API | 后端 |
| 工单列表页 + 详情页 | 0.5人日 | 前端页面 + 组件 | 前端 |
| 报工页面（触屏终端适配） | 1人日 | 报工表单 + 扫码组件 | 前端 |
| 报表页面 | 0.5人日 | ECharts 图表 + 表格 | 前端 |

**验收标准**：

- 扫码组件三级降级链：原生扫码 → jsQR → 手动输入
- 报表：个人日报 + 生产报表（日/周/月）

## 依赖关系

```
S1(认证+租户) ───────┬── S3(报工) ─── S4(看板)
                      │
S2(Excel导入) ────────┤
                      │
                      └── S5(消息+审批+字典) ──┬── S6(TPM)
                                                ├── S7(品质)
                                                └── S8(安灯)
```

> S1、S2 无相互依赖，可并行启动。S3 依赖 S1 的用户认证体系。S4 依赖 S3 的数据。S6~S8 依赖 S5 的消息中心和审批引擎。
