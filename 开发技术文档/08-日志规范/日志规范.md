# 日志规范

> **版本**：v1.0 | **适用范围**：知微 ziwi SaaS 后端服务

---

## 一、日志格式

所有日志输出为 **JSON 格式**（单行），便于日志中心（如 ELK、Loki）检索。

### 标准格式

```json
{
  "timestamp": "2026-06-12T10:00:00.000Z",
  "level": "INFO",
  "request_id": "req_a1b2c3d4e5f6",
  "tenant_id": "t_12345",
  "user_id": "u_67890",
  "trace_id": "trace_abc123",
  "module": "M01",
  "action": "work_report.submit",
  "message": "报工提交成功",
  "duration_ms": 120,
  "status_code": 200,
  "extra": {}
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `timestamp` | string | ✅ | ISO 8601 格式，UTC 时区，精度到毫秒 |
| `level` | string | ✅ | `DEBUG` / `INFO` / `WARN` / `ERROR` |
| `request_id` | string | ✅ | 每个 HTTP 请求唯一 ID，入口生成，贯穿请求生命周期 |
| `tenant_id` | string | 尽量 | 当前请求的租户 ID，中间件自动注入 |
| `user_id` | string | 尽量 | 当前操作用户 ID |
| `trace_id` | string | 链路追踪 | 跨服务调用链路的唯一 ID（MQ 消费/定时任务也生成） |
| `module` | string | 尽量 | 模块标识（`M01`~`M13`，按开发计划） |
| `action` | string | 尽量 | 业务动作（`{模块}.{操作}`，如 `work_order.create`） |
| `message` | string | ✅ | 人类可读的描述信息 |
| `duration_ms` | int | API 日志 | 请求处理耗时（毫秒） |
| `status_code` | int | API 日志 | HTTP 状态码 |
| `extra` | object | 按需 | 额外上下文字段（如 `order_id`, `error_stack`） |

## 二、日志级别规范

| 级别 | 使用场景 | 示例 |
|------|---------|------|
| **DEBUG** | 开发调试，默认关闭 | SQL 语句、临时变量值 |
| **INFO** | **业务操作记录**（默认级别） | 用户登录、报工提交、审批通过 |
| **WARN** | 异常但不影响主流程 | 重试第 1 次、请求参数异常但被降级处理、慢查询（>500ms） |
| **ERROR** | 需要人工介入的问题 | 数据库连接失败、第三方 API 调用失败、未捕获异常 |

### 分级原则

- **INFO**：记录"什么人、什么时间、做了什么"，可供业务审计追溯
- **WARN**：需要关注但系统可自愈，如：熔断半开、缓存失效、重试成功
- **ERROR**：必须告警，需要值班人员处理

## 三、request_id / trace_id 传递

### 3.1 request_id（请求级别）

- **生成时机**：每个 HTTP 请求到达 API 网关时
- **生成方式**：`uuid4()` 的简短形式（取前 20 字符）
- **传递方式**：
  - 后端内部：通过 `contextvars` 线程上下文传递
  - 响应头：返回 `X-Request-Id` 给前端
  - 日志：所有该请求相关的日志记录相同 `request_id`

### 3.2 trace_id（链路级别）

- **生成时机**：入口点（HTTP 请求/MQ 消费/定时任务启动）
- **生成方式**：`uuid4()` 简短形式
- **传递方式**：
  - 跨服务调用：在 HTTP Header 中传递 `X-Trace-Id`
  - MQ 消息：在消息头中携带 `trace_id`
  - 定时任务：每个任务执行周期生成新 `trace_id`

### 3.3 上下文传递（Python）

```python
from contextvars import ContextVar
import uuid

request_id_var: ContextVar[str] = ContextVar('request_id', default='')
trace_id_var: ContextVar[str] = ContextVar('trace_id', default='')

def generate_request_id() -> str:
    return f"req_{uuid.uuid4().hex[:16]}"

def generate_trace_id() -> str:
    return f"trace_{uuid.uuid4().hex[:12]}"
```

### 3.4 FastAPI 中间件示例

```python
from fastapi import Request
import time
import logging

logger = logging.getLogger("ziwi")

@app.middleware("http")
async def logging_middleware(request: Request, call_next):
    request_id = generate_request_id()
    trace_id = generate_trace_id()
    
    # 注入上下文
    request_id_var.set(request_id)
    trace_id_var.set(trace_id)
    
    start = time.monotonic()
    response = await call_next(request)
    duration_ms = int((time.monotonic() - start) * 1000)
    
    # 输出访问日志
    logger.info({
        "request_id": request_id,
        "trace_id": trace_id,
        "method": request.method,
        "path": request.url.path,
        "status_code": response.status_code,
        "duration_ms": duration_ms,
        "message": f"{request.method} {request.url.path} {response.status_code}",
    })
    
    response.headers["X-Request-Id"] = request_id
    return response
```

## 四、日志输出配置

### Python logging 配置

```python
import logging
import json

class JSONFormatter(logging.Formatter):
    def format(self, record: logging.LogRecord) -> str:
        log_entry = {
            "timestamp": self.formatTime(record, self.datefmt),
            "level": record.levelname,
            "request_id": request_id_var.get(),
            "tenant_id": tenant_id_var.get(),
            "user_id": user_id_var.get(),
            "trace_id": trace_id_var.get(),
            "module": getattr(record, 'module', ''),
            "action": getattr(record, 'action', ''),
            "message": record.getMessage(),
        }
        if record.exc_info:
            log_entry["exception"] = self.formatException(record.exc_info)
        return json.dumps(log_entry, ensure_ascii=False)

# 配置
logging_config = {
    "version": 1,
    "formatters": {"json": {"()": JSONFormatter}},
    "handlers": {
        "console": {
            "class": "logging.StreamHandler",
            "formatter": "json",
            "stream": "ext://sys.stdout",
        }
    },
    "root": {"level": "INFO", "handlers": ["console"]},
}
```

### 日志文件（生产环境）

- **标准输出**：所有日志写入 stdout/stderr（由 Docker/K8s 收集）
- **不写日志文件**：应用本身不落盘日志文件，由容器运行时处理
- **日志轮转**：由日志收集系统（如 Loki、Filebeat）负责

## 五、禁止行为

| 禁止项 | 说明 | 替代方案 |
|--------|------|---------|
| ❌ `print()` 调试 | 生产代码中禁止出现 `print` | 使用 `logger.debug()` |
| ❌ 记录密码/Token | 任何日志中不得出现明文密钥 | 记录 `***` 或 `user_id` |
| ❌ 循环中打日志 | for 循环内每轮都 `logger.info` | 汇总后单条记录 |
| ❌ 不区分级别 | 所有日志都用 `INFO` 或 `ERROR` | 按第二节的分级规范 |
| ❌ 日志内容过长 | 单条日志 > 10KB | 截断或放到 `extra` 中 |
