# 代码规范

> **版本**：v1.0 | **适用范围**：知微 ziwi SaaS 全栈开发

## 一、Python 后端规范（FastAPI）

### 1.1 代码风格

- 遵循 **PEP 8**，使用 **Black** 自动格式化（`line-length = 100`）
- 使用 **isort** 管理 import 顺序（先标准库 → 第三方 → 本地模块）
- 使用 **Ruff** 做 lint 检查（`pyproject.toml` 中统一配置）

### 1.2 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 类名 | `PascalCase` | `MultiTenantRepository`, `WorkReportService` |
| 函数/方法 | `snake_case` | `create_work_order()`, `get_user_by_id()` |
| 变量 | `snake_case` | `tenant_id`, `work_order_list` |
| 常量 | `UPPER_SNAKE_CASE` | `MAX_RETRY_COUNT`, `DEFAULT_PAGE_SIZE` |
| 模块名 | `snake_case` | `work_report.py`, `auth_service.py` |
| 私有函数/变量 | 前缀 `_` | `_validate_tenant()`, `_cache_key` |

### 1.3 类型注解

- 所有函数参数和返回值必须标注类型
- 复杂类型使用 `typing` 模块（`List`, `Dict`, `Optional`, `Union`）
- Pydantic model 用于请求/响应序列化

```python
from typing import Optional
from pydantic import BaseModel

class WorkOrderCreate(BaseModel):
    product_id: str
    quantity: int
    deadline: Optional[str] = None

async def create_work_order(
    order_data: WorkOrderCreate,
    tenant_id: str,
    user_id: str
) -> dict:
    ...
```

### 1.4 项目结构

```
backend/
├── app/
│   ├── api/          # 路由层（controller）
│   ├── core/         # 核心（config、security、db session）
│   ├── models/       # SQLAlchemy ORM 模型
│   ├── schemas/      # Pydantic 请求/响应 schema
│   ├── services/     # 业务逻辑层
│   ├── repositories/ # 数据访问层（DAL）
│   └── utils/        # 工具函数
├── tests/
│   ├── unit/
│   ├── integration/
│   └── conftest.py
├── alembic/          # 数据库迁移
├── pyproject.toml
└── requirements.txt
```

### 1.5 异常处理

- 业务异常使用 `HTTPException` + 统一错误码
- 不要吞异常，不要用裸 `except:`

```python
from fastapi import HTTPException

# ✅ 正确
if not user:
    raise HTTPException(status_code=404, detail={"code": "USER_NOT_FOUND", "message": "用户不存在"})

# ❌ 错误
try:
    ...
except:
    pass
```

## 二、Vue 3 前端规范

### 2.1 代码风格

- 遵循 [Vue 3 官方风格指南](https://vuejs.org/style-guide/)
- 使用 **ESLint** + **Prettier** 统一格式
- 使用 **TypeScript**，禁用 `any` 类型

### 2.2 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 组件名 | `PascalCase` | `WorkOrderTable.vue`, `AndonCallPanel.vue` |
| 文件名 | `PascalCase` | `UserProfile.vue`, `TenantSettings.vue` |
| 变量/函数 | `camelCase` | `userList`, `fetchWorkOrders()` |
| 常量 | `UPPER_SNAKE_CASE` | `MAX_FILE_SIZE`, `API_BASE_URL` |
| Props | `camelCase` | `:tenant-id="id"` → `props.tenantId` |
| Events | `kebab-case` | `@update:model-value` |

### 2.3 组件结构（Composition API + `<script setup>`）

```vue
<script setup lang="ts">
import { ref, onMounted } from 'vue'
import { useWorkOrderStore } from '@/stores/workOrder'

// Props
const props = defineProps<{
  tenantId: string
}>()

// Emits
const emit = defineEmits<{
  (e: 'submit', data: WorkOrderData): void
}>()

// State
const loading = ref(false)
const workOrders = ref<WorkOrder[]>([])

// Methods
async function fetchData() {
  loading.value = true
  try {
    workOrders.value = await api.getWorkOrders(props.tenantId)
  } finally {
    loading.value = false
  }
}

onMounted(fetchData)
</script>

<template>
  <div class="work-order-list">
    <div v-if="loading">加载中...</div>
    <div v-else>
      <WorkOrderCard v-for="item in workOrders" :key="item.id" :order="item" />
    </div>
  </div>
</template>

<style scoped lang="scss">
.work-order-list {
  padding: 16px;
}
</style>
```

### 2.5 API 调用规范


## 三、通用规范

### 3.1 注释

- 公共函数/类必须有 docstring（Python 用 Google 风格，JS 用 JSDoc）
- 复杂逻辑需写行内注释说明意图
- 不要写"显然"的注释（`i += 1  # 加1`）

### 3.2 错误处理

- 所有 API 调用必须处理错误状态（`try/catch` 或 `catch`）
- 前端统一使用 `ElMessage` / `ElNotification` 提示错误
- 后端统一错误响应格式：`{ "code": "ERROR_CODE", "message": "描述" }`

### 3.3 安全

- 禁止在前端代码中硬编码密钥/Token
- 禁止将用户输入拼接到 SQL/Shell 命令中
- 所有用户输入必须做 XSS 过滤（后端输出时转义）

### 3.4 性能

- 列表页必须分页，禁止一次加载全部数据
- 图片/文件上传必须限制大小（前端+后端双重校验）
- API 响应时间 > 500ms 的接口必须在设计阶段确认是否需要异步处理
