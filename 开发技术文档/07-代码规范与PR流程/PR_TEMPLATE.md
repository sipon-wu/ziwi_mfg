## 自检清单

### 代码质量

- [ ] 代码遵循项目代码规范
- [ ] 无硬编码敏感信息（密钥/密码/Token）
- [ ] 无 `console.log` / `print` 调试代码残留
- [ ] 无 `TODO` / `FIXME` 遗留（如有则在描述中说明）
- [ ] 日志记录完整（关键业务操作有 INFO 级别日志）
- [ ] 错误处理完善（所有外部调用有 try/catch）

### 安全性

- [ ] 用户输入已校验/转义
- [ ] API 鉴权已确认（需登录/需管理员/公开）
- [ ] 租户隔离已确认（数据不可跨租户访问）
- [ ] 不存在 SQL 注入风险

### 兼容性

- [ ] 前端向后兼容（不影响已发布功能）
- [ ] API 向后兼容（不修改已有接口签名）
- [ ] 数据库迁移可回滚

## 部署说明

### 发布注意事项

> 如有特殊部署顺序、数据迁移、功能开关等，在此说明

```
1. 先执行 `alembic upgrade head` 更新数据库
2. 设置环境变量 `FEATURE_WORK_REPORT_V2=true`
3. 重启后端服务
```

### 回滚方案

> 如果本次发布出现问题，如何回滚

```
1. FEATURE_WORK_REPORT_V2=false
2. alembic downgrade -1
3. 重启后端服务
```
