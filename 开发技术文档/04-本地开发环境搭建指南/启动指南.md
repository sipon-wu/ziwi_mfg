# 本地开发环境搭建指南

> **版本**：v1.0 | **适用平台**：Windows / macOS / Linux

---

## 一、环境要求

| 工具 | 版本要求 | 检查命令 | 备注 |
|------|---------|---------|------|
| Docker | ≥ 24.0 | `docker --version` | 含 Docker Compose |
| Docker Compose | ≥ 2.20 | `docker compose version` | 推荐 V2 版本 |
| Python | ≥ 3.13 | `python --version` | 本地运行后端时使用 |
| Node.js | ≥ 20 LTS | `node --version` | 本地运行前端时使用 |
| pnpm | ≥ 9 | `pnpm --version` | 前端包管理器 |
| Git | ≥ 2.40 | `git --version` | 代码版本管理 |

## 二、一键启动（推荐）

使用 Docker Compose 一键启动所有依赖服务 + 应用。

### 2.1 克隆仓库

```bash
git clone <仓库地址> ziwi
cd ziwi
```

### 2.2 配置环境变量

```bash
# 复制环境变量模板
cp 开发技术文档/04-本地开发环境搭建指南/.env.example .env

# 根据需要修改配置（默认值可直接用于本地开发）
# 如需连接外部服务（企微/钉钉/短信），填写对应配置项
```

### 2.3 启动所有服务

```bash
# 启动全部服务（数据库 + 缓存 + 对象存储 + MQTT + 后端 + 前端）
docker compose -f 开发技术文档/04-本地开发环境搭建指南/docker-compose.yml up -d

# 查看启动状态
docker compose -f 开发技术文档/04-本地开发环境搭建指南/docker-compose.yml ps

# 查看实时日志
docker compose -f 开发技术文档/04-本地开发环境搭建指南/docker-compose.yml logs -f
```

### 2.4 验证各服务

| 服务 | 访问地址 | 验证方法 |
|------|---------|---------|
| 后端 API | http://localhost:8000 | `curl http://localhost:8000/health` 返回 200 |
| 前端页面 | http://localhost:5173 | 浏览器打开，看到登录页 |
| PostgreSQL | localhost:5432 | `psql -U ziwi -d ziwi -c "SELECT 1"` |
| PgBouncer | localhost:6432 | `psql -U ziwi -d ziwi -p 6432 -c "SELECT 1"` |
| Redis | localhost:6379 | `redis-cli -a ziwi_redis_password ping` 返回 PONG |
| MinIO Console | http://localhost:9001 | 账号: `ziwi_minio_admin` / 密码: `ziwi_minio_password` |
| EMQX Dashboard | http://localhost:18083 | 默认 admin / `ziwi_emqx_password` |

## 三、服务启动顺序

依赖服务有严格的启动顺序要求，`docker compose up` 已自动处理，但手动启动时请注意：

```
① PostgreSQL + TimescaleDB
  └── ② PgBouncer（等待 PostgreSQL 就绪）
       └── ④ 后端 FastAPI（等待 PgBouncer + Redis + MinIO）
③ Redis（可与 ① 并行）
⑤ MinIO（可与 ① 并行）
⑥ EMQX（可与 ① 并行）
       └── ⑦ 前端 Vue 3（等待后端）
```

> Docker Compose 的 `depends_on` + `healthcheck` 已自动处理启动顺序。

## 四、本地开发（非 Docker）

### 4.1 启动依赖服务（仅 DB/Redis/MinIO/EMQX）

```bash
# 只启动基础设施，不启动后端和前端容器
docker compose -f 开发技术文档/04-本地开发环境搭建指南/docker-compose.yml up -d \
  postgres redis minio emqx pgbouncer
```

### 4.2 后端（本地运行）

```bash
cd backend

# 创建虚拟环境
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 数据库迁移
alembic upgrade head

# 启动开发服务器（热重载）
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### 4.3 前端（本地运行）

```bash
cd frontend

# 安装依赖
pnpm install

# 启动开发服务器（热重载）
pnpm dev
```

## 五、数据库初始化

### 5.1 自动初始化

首次启动时，PostgreSQL 容器会自动执行 `docker-entrypoint-initdb.d/` 目录下的 SQL 脚本创建基础表结构。

### 5.2 手动迁移

```bash
# 使用 Alembic 管理数据库版本
cd backend

# 创建新的迁移
alembic revision --autogenerate -m "描述变更"

# 应用迁移
alembic upgrade head

# 回滚迁移
alembic downgrade -1
```

## 六、环境变量清单

| 分类 | 变量名 | 必填 | 默认值 | 说明 |
|------|--------|------|--------|------|

## 七、常见问题

### Q1：端口冲突

> 错误信息：`port is already allocated`

```bash
# 检查端口占用
netstat -ano | findstr :5432  # Windows
lsof -i :5432                  # macOS/Linux

# 修改 docker-compose.yml 中对应服务的 ports 映射
```

### Q2：Docker 容器启动后马上退出

> 使用以下命令查看错误日志：

```bash
docker compose logs <service_name>
# 例如：docker compose logs postgres
```

### Q3：数据库连接失败

```
# 检查 PgBouncer 是否正常运行
docker compose ps pgbouncer

# 检查 PostgreSQL 健康状态
docker compose exec postgres pg_isready -U ziwi

# 尝试直连 PostgreSQL（跳过 PgBouncer）
psql -h localhost -p 5432 -U ziwi -d ziwi
```

### Q4：前端热更新慢

```
# 在 frontend/vite.config.ts 中启用
server:
  watch:
    usePolling: true  # Docker 环境需要文件轮询
```

### Q5：如何重置开发环境？

```bash
# 停止并删除所有容器和卷（数据会丢失！）
docker compose -f 开发技术文档/04-本地开发环境搭建指南/docker-compose.yml down -v

# 重新启动
docker compose -f 开发技术文档/04-本地开发环境搭建指南/docker-compose.yml up -d
```
