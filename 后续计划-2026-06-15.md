## 四、后续路线建议

### 路 A：补后端短板（推荐，今日可完成 ~3h）

```
Service 层补齐（审批/TPM/品质/安灯/能碳）→ 安灯前端页面 → 同步层消费者
```

**理由**：Service 层是模式化填空工作，最快产出；安灯页面是生产现场高频需求；同步层消费者接上后能碳同步闭环就完整了。

### 路 B：补前端页面

```
品质前端 → 安灯前端 → 能碳前端 → 数据采集前端
```

**理由**：14 个模块中只有 4 个模块有前端页面，缺口最大。但每个页面约 1-1.5h，5 个页面需要半天以上。

### 路 C：推进 IoT/配置入口大件

```
IoT 协议适配 → ziwi.cn 配置入口 → License 服务
```

**理由**：原始 Phase 1 计划缺口最大的两块。但每件 3-4h，今天做不完。
