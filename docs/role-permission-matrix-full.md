|---------|---------|-----------|---------|:------------:|

| `inspector` | 质检员 | DEPT（本组织及直属子组织） | 品质检验员 | ✅ |
| `viewer` | 只读用户 | DEPT（本组织及直属子组织） | 演示/参观 | ✅ |

## 3. 数据作用域说明

每个角色能"看到哪些数据"由其数据作用域决定：

| 级别 | 含义 | 举例（以工单为例） |
|------|------|-------------------|
| **ALL** | 全租户所有数据 | 能看到本租户下所有部门/车间/班组的工单 |
| **DEPT_CHILD** | 本组织及所有下级组织 | 生产部门主管能看到生产部+下属一车间/二车间+班组A/B/C/D的工单 |
| **DEPT** | 本组织及直属子组织 | 一车间小组长能看到一车间+班组A/班组B的工单 |
| **SELF** | 仅本人创建/负责的数据 | 操作员仅能看到自己报工的工单、自己创建的报工记录 |

### 各角色实际数据范围

| 角色 | 典型归属组织 | 数据范围 |
|------|------------|---------|
| admin | 租户根节点 | 全租户 |
| department_head | 部门（level=1） | 本部门 + 下属所有车间 + 下属所有班组 |
| team_leader | 车间（level=2） | 本车间 + 下属所有班组 |
| operator | 班组（level=3） | 仅本人 |
| scheduler | 部门（level=1） | 本部门 + 下属所有车间 |
| inspector | 品质部门（level=2） | 品质部门及其下属检验组 |
| viewer | 任意 | 所属组织及直属子组织（只读） |
