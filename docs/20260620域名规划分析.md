## 五、命名原则建议

```
【方案A】产品线缩写前缀（当前方案）
  教育: edu-xxx.ziwi.cn
  智造: 无统一前缀（cloud/admin/api/license 散落）
  问题: 智造命名不统一，两线不对等

【方案B】产品线标识前缀
  教育: school-xxx.ziwi.cn
  智造:  cloud-xxx.ziwi.cn
  优点: 清晰对等
  问题: school-demo.ziwi.cn 不简洁

【方案C】按角色/功能统一命名 + 产品线二级
  教育主站: edu.ziwi.cn     → 其子功能用路径 edu.ziwi.cn/demo
  智造主站: cloud.ziwi.cn   → 其子功能用路径 cloud.ziwi.cn/demo
  优点: 每个产品线只暴露一个域名给用户，简洁
  问题: 内部管理/API仍需独立域名，路径层次深

【方案D】产品线独立根域名 + 路径拆分
  教育: school.ziwi.cn/demo, school.ziwi.cn/admin, ...
  智造: cloud.ziwi.cn/demo, cloud.ziwi.cn/admin, ...
  优点: 极端简洁，用户只记一个域名
  问题: admin/api 需要独立容器时无法路径反代

【方案E】两条线 + 公共服务三区对称
                 教育          智造          共享
  主站           edu.ziwi.cn    cloud.ziwi.cn   www.ziwi.cn
  演示           E3             M5              —
  试用           E4             M6              —
  API            E6             M4              —
  管理后台       E5             M3              C5(统一入口)
  CDN            —              —               C1
  文档/帮助      —              —               C2/C3
  存储           —              —               C4
  心跳           —              —               C6
```

## 六、待你决策的问题

1. **主站域名**：教育用 `school.ziwi.cn` 还是 `edu.ziwi.cn`？哪个对教师更友好？
2. **管理后台**：两线各自独立域名，还是统一用一个 `ops.ziwi.cn`？
3. **demo/trial**：独立域名（`demo.ziwi.cn` 只给教育），还是路径式（`school.ziwi.cn/demo`）？
4. **智造租户**：`{tenant}.cloud.ziwi.cn` 这种 Wildcard 是否确认？
5. **API 命名**：`edu-api.ziwi.cn` / `api.ziwi.cn`，还是都带上产品线前缀更平等？
6. **Phase 1 vs Phase 2**：哪些域名现在就需要，哪些可以等？
